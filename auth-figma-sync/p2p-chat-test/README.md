# P2P Chat Test Prototype

## Что внутри
- `index.html` — кликабельный прототип экрана активной сделки с чатом.
- `styles.css` — финальный визуальный стиль (палитра Graphite Mint).
- `app.js` — интерактивные состояния: offline/online, синхронизация, тяжелый сценарий.
- `tokens.json` — базовые дизайн-токены для переноса в Figma Variables.

## Как открыть
1. Открой `auth-figma-sync/p2p-chat-test/index.html` в браузере.
2. Либо запусти локальный сервер из корня проекта:
   - `python3 -m http.server 4173`
   - открой `http://localhost:4173/auth-figma-sync/p2p-chat-test/`

## Что показывать на защите по задачам

### Задача 1
- Базовый экран активной сделки: header, SLA, статус, чат, панель деталей.

### Задача 2
- Нажми `Смоделировать Offline`.
- Отправь сообщение: оно станет `Отправляется (локально)`.
- Нажми `+30с событий` (симуляция входящих, edit/delete/error).
- Нажми `Вернуться Online` для reconciliation.

### Задача 3
- Нажми `Нагруженный экран`.
- Получишь микс типов сообщений: user/system/error/important/pending/deleted.
- Прокрути список вверх, увидишь `Новых сообщений: N ↓`.
- Протестируй фильтры `Системные / Ошибки / Важные`.

### Задачи 4–6
- Обосновываются текстом из решения (MVP, риски, путь сообщения).
- Прототип подтверждает ключевые UX-механики: статусы, queue, sync, контроль скролла.

### Задача 7
- Иконки и UI-элементы: SVG/вектор.
- Аватары и пользовательские изображения: raster.

### Задача 8
- Визуал применен в `styles.css` и `tokens.json`.

## Как перенести в Figma бесплатно

### Вариант A (быстро)
1. В Figma создай Frame 1440x1024.
2. Перенеси токены из `tokens.json` в Variables (цвета, радиусы, spacing).
3. Собери layout Auto Layout-блоками по структуре экрана:
   - `deal-header`
   - `chat-column`
   - `right-rail`
4. Для message item сделай компонент с вариантами:
   - `incoming`, `outgoing`, `system`, `error`, `important`, `pending`.

### Вариант B (через твой plugin flow)
1. Открой `auth-figma-sync/figma-plugin` как Development plugin.
2. Используй его как базу генерации фреймов.
3. Стайлгайд бери из `tokens.json`, а контент и состояния — из `app.js`.

## Ограничения, учтенные в демо
- Web only.
- Polling вместо WebSocket.
- Без большого локального кэша (только очередь/видимая лента).
- Подготовлено к виртуализации (фиксированная зона списка и простые item-типы).
- Без sticky внутри сообщений.
